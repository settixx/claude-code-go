# Verification Skill

Verify that an implementation works correctly.

## Instructions

1. **Read the requirements**: Understand what the code is supposed to do.
2. **Run existing tests**: Execute the test suite and note any failures.
3. **Manual validation**: If no tests exist, run the code with representative inputs.
4. **Edge cases**: Test boundary conditions, empty inputs, error paths.
5. **Compilation check**: Ensure the code compiles/lints cleanly.
6. **Report**: Summarize pass/fail status and any issues found.

If tests are missing for critical paths, suggest adding them.
